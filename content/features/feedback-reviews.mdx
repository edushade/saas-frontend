faq:
  title: "Feedback & Reviews FAQ"
  description: "How ratings, written reviews, moderation, notifications and the satisfaction dashboard work inside Edushade."
  items:
    - question: "What can a learner review in Edushade?"
      answer: "Five entity types share the same review system: courses, learning paths, cohorts, live events and quizzes. The shape of a review — 1–5 rating, written body, status, creator, audit fields — is identical across all of them, so reporting and moderation behave the same everywhere."
    - question: "Who is allowed to leave a review?"
      answer: "Reviews are gated by enrollment. The platform checks that the learner is actually enrolled in the program before accepting the submission, which keeps drive-by ratings out of the dataset. Admins can post a review on behalf of someone when needed — those skip the moderation queue and publish on save."
    - question: "What is the moderation lifecycle?"
      answer: "Three statuses: pending, active and archived. Learner submissions start as pending. An admin moves them to active to make them visible on the public site, or to archived to retire without losing the record. When a learner edits their own review, the status drops back to pending so the change is re-checked."
    - question: "How do reviews behave on cohort-based courses?"
      answer: "If a learner is enrolled in a cohort under a course or learning path, the review is stored against the cohort automatically — so reviews track the actual batch they took, not the parent template. Aggregations roll cohort reviews back up to the parent program when reporting at that level."
    - question: "Can an instructor disable reviews on a single program?"
      answer: "Yes. Each course and learning path carries a rating-enabled flag, on by default. Switching it off keeps the review form from rendering on that program without affecting any other course on the platform — useful for sensitive content, preview programs, or pilots."
    - question: "What does the learner see on a program page?"
      answer: "An average rating in the form of X.X/5, a star row, a distribution bar for 5/4/3/2/1 ratings, the total review count, and a Write a Review button. If the learner has already reviewed that program, the button shows Already Reviewed and does not open the form again."
    - question: "How does the admin manage reviews?"
      answer: "Each program in the admin has a Reviews tab with a data table — reviewer name and avatar, rating as a star row, status badge (pending/active/archived), comment, review date, plus row-level edit and delete. Bulk actions cover CSV export and bulk delete, and there is a Create Review action for admin-side submissions."
    - question: "Are notifications sent when reviews come in?"
      answer: "Yes — both directions are wired up. review.received fires to the instructor when a new review lands; review.response_received fires to the learner when the instructor responds. Both ship as in-app, email, SMS and push templates, all editable from the notification settings UI."
    - question: "What aggregate metrics ship with reviews?"
      answer: "Average rating, total reviews, current-month vs previous-month rating growth, a six-month satisfaction line (avg_rating / 5 × 100, per month), and a batch average-ratings endpoint that returns ratings + counts for up to 1,000 entities in one call. Course cards, path cards and the satisfaction dashboard all read from this data."
    - question: "Can reviews be exported?"
      answer: "Yes. The admin Reviews table exposes a CSV export with full metadata — reviewer ID, entity type and ID, rating, status, body, created and updated timestamps, plus the audit fields — so reviews can be backed up, audited or moved into another system whenever you need to."
    - question: "What controls who can manage reviews on the admin side?"
      answer: "The platform ships with five RBAC permissions for reviews: review.list, review.view, review.create, review.update and review.delete. Assign them to a role and any user with that role inherits the access, scoped to the tenant. Course and cohort scoping comes from the existing program permissions."
    - question: "How long can a review body be?"
      answer: "Up to 10,000 characters, with a minimum of 1 character — both rating and body are required to submit. That is enough room for a real piece of feedback, not just a one-liner, and the form uses a textarea so longer responses are easy to write and edit."
