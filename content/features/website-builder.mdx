faq:
  title: "Website Builder FAQ"
  description: "How the Edushade website builder works — pages, blocks, themes, locales, versions, previews and how it integrates with the rest of the platform."
  items:
    - question: "What is the website builder in Edushade?"
      answer: "It is a block-based content system built into the platform — pages, posts, forms, menus, themes, locales and versions all live in the same admin alongside your courses. Output is a real public site for the tenant: marketing pages, blog, contact forms, navigation, the lot."
    - question: "How many block types are there?"
      answer: "Forty base block types — heading, paragraph, image, video, hero, columns, grid, gallery, accordion, tabs, form, table, code, chart, timeline, stats, quote, rating, list, link, button, card, carousel, modal, lightbox, embed, audio, divider, spacer, stack, flex, container, dropdown, progress, section, row, column, tooltip and a couple more. Each one has its own style controls, responsive overrides and animation settings."
    - question: "Are there learning-specific blocks?"
      answer: "Yes. On top of the base blocks, the platform ships templated blocks built for an academy: course detail layouts, course list grids, cohort selectors, cohort schedules, cohort countdowns, event blocks, curriculum trees, enrollment panels, pricing grids, FAQ blocks, instructor lists, testimonial sections and more — pre-wired to the underlying course, cohort and event data."
    - question: "Can I run the site in multiple languages?"
      answer: "Yes. Add as many locales as you want — each with a code, label, flag emoji, RTL flag and an optional fallback locale. Page titles, meta and content fields are localized per record, so the same page serves the right language without duplicate rows."
    - question: "Does it support right-to-left languages?"
      answer: "Yes. Set is_rtl on the locale and the site renders right-to-left for that language. Combined with per-locale typography overrides, an English landing page can run on Inter while the Arabic version runs on a script-appropriate font without compromise."
    - question: "How does the theming work?"
      answer: "Theme settings live on the tenant. Define a light palette in OKLCH (required) and a dark palette (optional), set a border-radius scale between 0 and 2, pick fonts for sans, serif and mono with letter-spacing per role, and choose whether the default theme should be light, dark or system-controlled."
    - question: "Can pages have different headers or footers?"
      answer: "Yes. The site has a default header, footer and announcement bar, and any page can either point to a different one (per-page IDs) or hide each of the three independently. A landing page can run header-only; a checkout page can run none of the three."
    - question: "How are versions handled?"
      answer: "Every page and post is versioned automatically. Versions carry a number, an optional name, optional notes, and an auto_save flag that separates background checkpoints from named saves. Roll back to any version, compare two versions side by side, and the live page is always the latest published one."
    - question: "Can I share a draft for review without publishing?"
      answer: "Yes — generate a preview. Each preview snapshot stores its own data, supports an optional password and an optional expiry timestamp, and can be flagged public or kept private. Stakeholders open the link without seeing the rest of the platform."
    - question: "Does the builder include a blog?"
      answer: "Yes. Posts have a slug, localized title, excerpt and content, a featured image, an author, tags and the same draft / published / archived status pipeline as pages. Posts version the same way, preview the same way, and publish through the same flow."
    - question: "What about forms?"
      answer: "Forms are first-class records — name, slug, localized title and description, a configurable fields array, settings JSON, an active / inactive status, and a submissions table that counts entries per form. Drop a form block into any page or wire it directly into a flow."
    - question: "Can I build navigation menus?"
      answer: "Yes. Menus are stored on the platform with nested items, and the site reads them through the header and footer blocks. Build a primary navigation, a footer link list, or both — the same nesting model handles a single-level header and a two-level mega menu."
    - question: "What states does a page move through?"
      answer: "Three: draft (in progress), published (live on the public site) and archived (retired but kept on file). Posts share the same pipeline. Forms use a separate two-state model — active and inactive — since a form does not have a publish moment in the same way a page does."
    - question: "Can I export the page or back it up?"
      answer: "The version history is the backup — every save, manual or auto, is on the record with its data snapshot. Roll back at any time, or compare two versions to see exactly what changed. For a full tenant export, the site settings, pages, posts, menus and forms are all retrievable from the website-service API."
    - question: "Does the builder integrate with the rest of the platform?"
      answer: "Yes. The course, cohort, event and product blocks read live data from the platform — so a course detail block on a landing page surfaces the published curriculum, enrollment panel, pricing and instructors of an actual course, not a static copy. Update the course; the site updates with it."
