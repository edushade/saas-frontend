faq:
  title: "Drip Content FAQ"
  description: "How drip schedules work in Edushade — strategies, overrides, lock rules and per-learner re-scheduling across courses, learning paths and cohorts."
  items:
    - question: "What is drip content in Edushade?"
      answer: "Drip is the rule that decides when a curriculum item — section, lesson, quiz, assignment or downloadable file — becomes available to a learner, and optionally when it locks again. Each rule lives next to the item it controls and binds to the program (course, learning path or cohort) it belongs to."
    - question: "Which programs can use drip?"
      answer: "All three program types: courses, learning paths and cohorts. The drip engine is the same across them, so the rule shapes you learn for a course apply to a cohort and a learning path without anything new to remember."
    - question: "What unlock modes are supported?"
      answer: "Four: Immediate (no rule, content opens on enrollment), Fixed Date (opens on a specific date and time), Enrollment Offset (opens a number of days after the learner enrolls) and Completion-Based (opens after a prerequisite curriculum item is completed). Per-item rules can use any of the four; the program-level strategy exposes Immediate, Fixed Date, Completion-Based and Custom."
    - question: "What lock modes are supported?"
      answer: "Four: No Lock, Fixed Date, Enrollment Offset (lock a number of days after enrollment) and Duration After Unlock (lock a number of days after the item became available to that specific learner). Lock is optional — pair it only with the items that need a closing window."
    - question: "Can I set drip rules per item or only program-wide?"
      answer: "Both. Pick a program-level strategy first — Immediate, Fixed Date, Completion-Based or Custom — and that becomes the default for every item. Then override individual items when the program rule does not fit. Custom mode means every item carries its own rule from the start."
    - question: "Which content types can carry a drip rule?"
      answer: "Five: section, lesson, quiz, assignment and file. A section drip applies to the section as a whole; the items inside follow the program rule unless they have their own override."
    - question: "How does completion-based unlock work?"
      answer: "On a completion-based item, you pick another curriculum item from the same program as the prerequisite. The item stays locked until the learner completes the prerequisite, then unlocks for them. Each learner progresses on their own timeline because completion is tracked per learner."
    - question: "Can I edit drip rules in bulk?"
      answer: "Yes. Switch the curriculum tree into bulk mode, select the items that should follow the same rule, and apply unlock and lock settings once. The same dialog handles single-item edits, so the experience is consistent whether you are touching one item or twenty."
    - question: "What happens when a learner hits a locked item?"
      answer: "They see a locked badge with the right reason — a release date for fixed-date and enrollment-offset rules, the prerequisite name for completion-based rules, or a closed badge if the lock window has passed. Learner-side messaging is automatic; you do not write copy for each rule."
    - question: "Can I give a single learner a different schedule?"
      answer: "Yes. User-level overrides let you re-schedule a specific item for a specific list of learners — new unlock date, new lock date, only for those users. The default schedule keeps running for everyone else, and the override is listed in its own panel so it stays visible and editable."
    - question: "How are user-level overrides different from item-level overrides?"
      answer: "An item-level override changes the rule for that item across every learner in the program. A user-level override changes the rule for that item only for the specific learners you select. Item-level handles 'this lesson is different'; user-level handles 'this learner needs a different window'."
    - question: "Do drip rules work with cohort schedules?"
      answer: "Yes. A cohort can carry its own drip rules on top of the bound course or learning path, so two cohorts can run the same curriculum on different rhythms — one strict weekly, one accelerated — without duplicating content. The cohort's rules apply only to learners enrolled in that cohort."
    - question: "Are drip rules counted in audit history?"
      answer: "Yes. Every drip schedule carries created-by, updated-by, created-at and updated-at fields, plus soft-delete on removal. Changes to overrides are attributable, and a removed override falls back to the next rule up — item-level if there is one, otherwise the program-level strategy."
    - question: "Do the drip rules show up in reports?"
      answer: "The drip configuration lives in the curriculum builder; learner state — unlocked, locked, closed and pending-prerequisite — is reflected in the curriculum sidebar and learner progress views. The schedule itself is exportable as part of the program metadata when needed."
