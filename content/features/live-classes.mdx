faq:
  title: "Live Classes FAQ"
  description: "How live classes work in Edushade — providers, scheduling, joining, attendance and how they relate to the rest of the curriculum."
  items:
    - question: "What is a live class in Edushade?"
      answer: "A lesson inside a course with lesson_type set to 'live'. It sits in the curriculum tree alongside video, audio, text, PDF and iframe lessons, and carries its own meeting provider, meeting URL, start time and optional password — but it is not a separate entity, so the same drip rules, prerequisites and progress engine all apply."
    - question: "How is a live class different from a workshop?"
      answer: "A workshop is a standalone, time-bound session with its own registration, capacity and participant list — independent from any course. A live class is part of a course curriculum, runs for the learners enrolled in that course, and contributes to course progress and completion. Use a workshop for a one-off; use a live class when synchronous instruction is part of a larger learning track."
    - question: "Which meeting providers are supported?"
      answer: "Google Meet, Zoom, Microsoft Teams and a generic 'Other' option for any URL-based provider. The provider is picked per lesson, so a single course can run different sessions on different platforms without changing anything platform-wide."
    - question: "Can I password-protect a live session?"
      answer: "Yes. Toggle password_protected on the lesson and add a meeting password. Enrolled learners see the password with a show/hide toggle on their lesson card; everyone else gets nothing."
    - question: "How does scheduling work?"
      answer: "Set meeting_starts_at as a date-time with timezone awareness. Edushade renders the countdown in the learner's locale and gates the join button to the meeting window. An optional duration lets the card display a finish time so learners can plan around it."
    - question: "How is attendance recorded?"
      answer: "When a learner clicks join, the useJoinLiveClass flow records the event server-side, marks the lesson in_progress with a started_at timestamp, and writes the join time into the learning_progress table. On session close, the lesson auto-completes — no separate attendance roster, no manual ticking."
    - question: "Can a single course mix live and recorded lessons?"
      answer: "Yes. The same course tree holds video, audio, text, PDF, iframe and live lessons together. A typical week could be a recorded video, a live discussion, a downloadable worksheet and a quiz — all inside one curriculum, all tracked by the same progress engine."
    - question: "Do drip rules apply to live classes?"
      answer: "Yes. Live lessons are curriculum items, so drip rules — fixed date, enrollment offset, completion-based or manual — work the same way they do for videos. Use a completion-based rule to require a prerequisite quiz before the live class unlocks, or a fixed-date rule to release the session at the start of a cohort week."
    - question: "Can I assign multiple instructors to one live class?"
      answer: "Yes. The instructor_ids field accepts an array of platform users, so a primary host and a guest expert can be assigned to the same lesson. The list surfaces on the learner side so attendees know who they are joining."
    - question: "Is the recording handled automatically?"
      answer: "The recording itself happens in the meeting provider — Zoom, Meet, Teams. Once you have the recording URL, you can either update the live lesson with the link, or attach a separate video lesson next to it as the on-demand follow-up. The follow-up plays in Edushade's own video player with full progress tracking."
    - question: "What happens if a learner misses the live class?"
      answer: "The lesson stays in not_started state on their progress record. If you have configured a recording or attached a follow-up video lesson, they consume that asynchronously — the same drip and prerequisite rules apply. Course completion logic considers the lesson complete based on whichever path you have set up."
    - question: "Can a live class be free or paid?"
      answer: "Pricing applies at the course level, not per lesson. The live class participates in whatever pricing the parent course uses — free, one-time, installments or part of a bundle. For a paid standalone live session, use a Workshop instead, which has its own pricing layer."
    - question: "Does the platform integrate directly with the meeting providers?"
      answer: "Edushade supports Google Meet, Zoom, Microsoft Teams and any other URL-based provider by capturing the meeting URL on the lesson. The session itself runs in the provider; Edushade is where the schedule, password, attendance and progress live."
    - question: "Can I export attendance for a live class?"
      answer: "Live class attendance lives on the learning_progress table, the same as every other lesson type. Pull progress records by entity_type='lesson' and entity_id=<lesson_id> to get the full attendance, with started_at, completed_at and time_spent for every learner."
