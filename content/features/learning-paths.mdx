faq:
  title: "Learning Paths FAQ"
  description: "Specifics about how learning paths, course-unlock logic, drip rules, certificates and cohorts work together inside Edushade."
  items:
    - question: "What is a learning path in Edushade?"
      answer: "A learning path is a programme wrapper around two or more existing courses. It carries its own title, summary, thumbnail, completion message, certificate and pricing — but the courses inside keep their own curriculum, assessments and instructors. Learners enroll in the path once, and Edushade tracks them across every course it contains."
    - question: "How is a learning path different from a single course?"
      answer: "A course holds a curriculum tree of sections, lessons, quizzes, assignments and files. A path holds courses. Use a course when the content is one continuous module, and a path when you want to bundle multiple courses — for example an onboarding programme, a multi-stage certification track or a role-based curriculum."
    - question: "How does course unlock work inside a path?"
      answer: "When a learner completes one course, Edushade fires a course-unlock notification and opens the next course in the sequence. You can layer drip rules on top — fixed dates, days after enrollment, prerequisite completion or manual unlocks — so progression is exactly as strict or as flexible as you need."
    - question: "Can a learning path be self-paced and cohort-based?"
      answer: "A path runs as either self-paced or cohort-based, the same way a course does. Self-paced means evergreen enrollment and learner-controlled pacing. Cohort-based means a scheduled batch with start and end dates, weekly live class days and times, late-enrollment toggle and a max enrollment cap."
    - question: "Are certificates issued at the path level?"
      answer: "Yes. Turn the certificate toggle on for the path, and Edushade issues one when a learner completes the entire programme. The certificate carries a unique number and a public verification code, with optional expiration. Individual courses inside the path can still issue their own certificates if you want both."
    - question: "How is progress tracked across multiple courses?"
      answer: "Edushade tracks progress percentage and score percentage at the path level, alongside last accessed at, last lesson and current grade rolled up across the courses inside. Programme managers see one row per learner per path, instead of stitching together per-course reports."
    - question: "Can I assign instructors to a learning path?"
      answer: "Yes. Edushade supports instructor, co-instructor and assistant roles at the path level — and the same roles continue to work inside each course in the path. A featured instructor can be highlighted on the path landing page."
    - question: "Can a learning path be sold?"
      answer: "Yes. Paths integrate with Edushade billing the same way courses do — one-time price, subscription, installment plan, or free. You can also include paths in a bundle alongside courses, cohorts and other products, and apply coupons or automatic offers to them."
    - question: "What states does a learning path go through before learners see it?"
      answer: "Paths move from draft to published, with archived as an additional state. The published_at timestamp is recorded automatically. Learners see paths only once they are published and enrollment is open."
    - question: "Can I export learning path data?"
      answer: "Yes. Edushade ships a CSV export of all learning path data — metadata, component courses, enrollments, progress and certificate records — so you can back up, audit, or move data into another system whenever you need to."
